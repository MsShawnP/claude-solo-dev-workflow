# Quick Start

Someone shared this with you because they think it'll help you build
software — even if you've never done it before. Here's what you need
to know.

---

## What this is (30-second version)

This is a system that makes an AI (Claude) help you build things the
right way. It gives you a step-by-step process so you don't have to
figure out what to do next — it tells you.

The three things it does:
1. **Keeps you on track** — tells you what to do next, catches you
   when you drift
2. **Remembers for you** — saves what happened each session so you
   can pick up tomorrow without forgetting anything
3. **Catches mistakes early** — reviews your work before you ship it

---

## Getting set up (15 minutes)

Read `START-HERE.md` in this same folder. It walks you through
installing everything step by step. You need:
- A terminal (your computer already has one)
- Git (free, 2-minute install)
- Claude Code (one command to install)

---

## The 3 commands that matter

Once you're set up, you really only need to know three things:

1. **`/init`** — Run this once to start a project. It creates your
   files and tells you what to do next.

2. **`/log`** — Run this after you finish something. It saves a
   checkpoint.

3. **`/wrap`** — Run this when you're done for the day. It captures
   everything so next time you can pick up exactly where you left
   off.

That's it. There are more commands, but these three are the
foundation. Claude will suggest the others when they're relevant.

---

## What to expect

- Claude will ask you questions before you start building. This is
  normal — it's making sure it understands what you actually want.
- Claude will nudge you to save checkpoints and wrap up sessions.
  Listen to it — this is how you avoid losing work.
- If you get lost, type `/commands` to see everything available, or
  just ask "what should I do next?"

---

## If something goes wrong

- Tell Claude what happened. It's good at fixing things.
- If Claude Code itself isn't working, close it and reopen it.
- If you're totally stuck, ask the person who shared this with you.

---

## One more thing

You don't need to understand how any of this works under the hood.
The whole point is that Claude handles the complexity while you focus
on what you're trying to build. Start with `/init`, follow the
prompts, and you'll figure it out as you go.
